# 算法题审题与解题方法论

> 目标：拿到一道算法题，从"毫无头绪"到"写出最优解"，建立一套稳定的思考框架。

---

## 第一章：审题 — 前 5 分钟的决策框架

### 1.1 三步审题法

拿到题目后，按以下顺序提取信息：

#### 第一步：圈定输入规模 → 反推允许的复杂度

这是 **最重要的审题动作**。题目的数据规模直接决定了你能用什么算法：

| n 的范围 | 允许的最高复杂度 | 常见算法 |
|-----------|-----------------|----------|
| n ≤ 10 | O(n!), O(2^n) | 回溯、暴力全排列、状态压缩 DP |
| n ≤ 20 | O(2^n) | 子集枚举、位运算枚举 |
| n ≤ 100 | O(n³) | Floyd-Warshall（多源最短路）、区间 DP |
| n ≤ 1,000 | O(n²) | 朴素 DP、双重循环、插入排序类 |
| n ≤ 10⁵ | O(n log n) | 排序、堆、二分、分治、平衡树 |
| n ≤ 10⁶ | O(n) | 哈希表、双指针、前缀和、单调栈、BFS/DFS |
| n ≤ 10⁸ | O(log n) 或 O(1) | 数学公式、位运算、二分 |

**为什么这很重要？** 如果 n ≤ 10⁵，你直接排除 O(n²) 的暴力解，思考方向立刻收窄到 O(n log n) 或 O(n)，避免浪费时间。

#### 第二步：识别数据结构关键词

题目描述中往往藏着数据结构的暗示：

| 题目中出现... | 优先考虑... |
|---------------|------------|
| "连续子数组" | 滑动窗口、前缀和 |
| "子序列"（不要求连续） | 动态规划 |
| "前 K 个" / "第 K 大" | 堆（优先队列）、快速选择 |
| "所有可能" / "所有组合" | 回溯 |
| "最短路径" / "最少步数" | BFS |
| "是否存在" / "能否到达" | DFS / BFS / 并查集 |
| "最大/最小" | 贪心（单步最优能推导全局最优）或 DP（需要多阶段决策） |
| "区间" / "重叠" | 排序 + 贪心、差分数组 |
| "O(1) 时间获取最小值" | 单调栈、最小栈 |
| 数据已排序 | 二分查找、双指针 |
| 树 / 链表 | DFS/BFS、递归、迭代 |
| 图 + "连通" | 并查集、DFS、BFS |

#### 第三步：锁定返回值要求

- **返回索引** → 不能排序（失去原始索引）
- **返回布尔值** → 可能可以提前终止
- **返回所有解** → 必须完整遍历，通常是回溯
- **返回一个最优解** → 贪心或 DP

### 1.2 审题自检清单

读完题后，用 5 个问题快速自检：

1. **数据规模多大？** → 反推复杂度上限
2. **数据有规律吗？** → 有序？有重复？有负数？全是正数？
3. **要求连续吗？** → 决定是子数组（滑动窗口/前缀和）还是子序列（DP）
4. **返回值是什么类型？** → 索引/布尔/所有解/最优值
5. **有没有特殊条件？** → 比如"只会有一个答案"、"数组非空"这类简化条件

## 第二章：8 大解题模式识别表

算法题的本质是 **模式匹配**。你把题目特征和这 8 个模式对号入座，就能快速定位解法方向。

### 模式 1：哈希表 — 以空间换时间

**识别信号**：
- 需要"快速查找是否存在"某个值
- "两数之和"类配对问题
- 需要"去重"或"计数"
- 需要"记录位置"（值 → 索引映射）

**代表题目**：两数之和、字母异位词分组、最长连续序列

**关键思维**：当暴力解法需要内层循环查找时，用 HashMap 把 O(n) 查找降为 O(1)。

### 模式 2：双指针 — 利用有序性或对称性

**识别信号**：
- 数组 **已排序**（或可以排序而不影响答案）
- 需要"原地修改数组"（移动零、删除重复项）
- "盛最多水"类两端收缩问题

**代表题目**：移动零、盛最多水的容器、三数之和

**关键思维**：两个指针从两端或同向移动，每次排除一个元素，O(n²) → O(n)。

### 模式 3：滑动窗口 — 处理连续子数组/子串

**识别信号**：
- 题目含"连续"二字
- "最长/最短子串满足条件"
- 涉及"无重复字符"、"覆盖子串"

**代表题目**：无重复字符的最长子串、找到字符串中所有字母异位词

**关键思维**：维护一个 [left, right) 窗口，right 扩展，left 在条件不满足时收缩。每个元素进一次出一次，O(n)。

### 模式 4：动态规划 — 多阶段最优决策

**识别信号**：
- "最长/最短/最大/最小" + "子序列"（不要求连续）
- "有多少种方法/路径"
- 问题的解可以由子问题的解推导（最优子结构）
- 当前决策依赖于之前的状态

**代表题目**：爬楼梯、最长递增子序列、打家劫舍

**关键思维**：
1. 定义 dp[i] 的含义（最重要的一步）
2. 找到状态转移方程
3. 确定初始化条件和遍历方向

### 模式 5：回溯 — 穷举所有可能性

**识别信号**：
- "所有可能解" / "所有组合" / "所有排列"
- "找出所有满足条件的子集"
- 需要尝试 → 撤销 → 换一条路

**代表题目**：全排列、子集、括号生成

**关键思维**：递归树遍历。在递归前做选择，递归后撤销选择（状态恢复）。配合剪枝优化。

### 模式 6：BFS/DFS — 图和树的遍历

**识别信号**：
- 明确给出一棵树或图
- "最少步数" / "最短路径" → BFS
- "是否存在路径" → DFS 或 BFS
- 二维矩阵的"连通块"问题 → DFS（沉岛法）

**代表题目**：岛屿数量、二叉树的层序遍历

**关键思维**：BFS 用队列（层序遍历，求最短路径）；DFS 用递归或栈（先走到底再回溯）。

### 模式 7：单调栈 — 寻找下一个更大/更小元素

**识别信号**：
- "下一个更大的元素"
- "柱状图中最大的矩形"类面积问题
- "每日温度"类

**代表题目**：每日温度、柱状图中最大的矩形

**关键思维**：栈内元素保持单调。当遇到违反单调性的元素时，不断弹栈并计算。每个元素入栈出栈各一次，O(n)。

### 模式 8：二分查找 — 有序空间中的快速定位

**识别信号**：
- 数组已排序
- "搜索" + 数据有序
- 答案在一个范围内，可以"猜答案然后验证"（二分答案）

**代表题目**：搜索旋转排序数组、在排序数组中查找元素的第一个和最后一个位置

**关键思维**：每次排除一半。不只是找具体值，还可以二分"满足条件的最小值"（边界二分）。

## 第三章：从暴力到最优 — 4 步推导法

### Step 1：先写出暴力解（哪怕 O(n²) 甚至 O(2ⁿ)）

不要跳过这一步。暴力解的作用：
- 帮你理解问题的本质约束
- 提供正确的输出，用于验证后续优化
- 让你看清楚哪里在"做无用功"

### Step 2：找到暴力解中的"重复计算"

问自己：暴力解循环中，有没有某个值被反复计算？有没有某个查找可以缓存？

```
暴力：对于每个元素 a[i]，循环查找是否存在 target - a[i]
      → 内层循环 O(n)，总 O(n²)
发现：查找操作是重复的，可以用 HashMap 缓存 → O(1) 查找，总 O(n)
```

### Step 3：找"单调性"或"排除性"

很多优化来自一个洞察：**有些数据可以不用看**。

```
双指针：如果 a[left] + a[right] > target，那么 right 左边的所有元素都不可能和 a[right] 配对了
       → right--，一次排除多个候选
```

### Step 4：检查是否可以用数学直接计算

少数题目有数学规律（如约瑟夫环、Nim 游戏），直接 O(1) 公式，不需要模拟。

## 第四章：代码实现 Checklist

写完代码后，逐项检查：

### 4.1 边界条件
- [ ] 空输入 / 单元素输入
- [ ] n = 0, n = 1
- [ ] 数组中全是相同值
- [ ] 数组中全是负数/正数

### 4.2 循环与控制
- [ ] while 循环的终止条件（不会死循环？）
- [ ] for 循环的索引边界（不会越界？）
- [ ] 递归的 base case（有终止条件吗？）

### 4.3 数据结构
- [ ] HashMap 存什么 key → 什么 value？
- [ ] 栈 / 队列在操作前判空了吗？
- [ ] 指针移动前判空了吗？

### 4.4 手动跑一个最小用例
在脑中或用纸笔，跑一遍最简单的输入。80% 的 bug 在这一步会被发现。

## 第五章：刷题策略建议

### 5.1 按分类刷，不要按题号刷
刷完一类再刷下一类。同类题目的解法结构相似，连刷 5 题双指针比零散地刷 5 题不同类的效果好得多。

### 5.2 每道题的三遍法
1. **第一遍（30 分钟）**：自己思考并写代码，不查题解
2. **第二遍（20 分钟）**：看题解，对比差距，记录"为什么最优解这么设计"
3. **第三遍（第二天，10 分钟）**：不看题解，重写最优解。能写出 80% 以上才算掌握

### 5.3 复杂度估算速查

| 数据量 n | 推荐算法时间复杂度 | 不推荐 |
|----------|-------------------|--------|
| n ≤ 15 | O(2ⁿ) 回溯 | — |
| n ≤ 100 | O(n³) | O(2ⁿ) |
| n ≤ 1,000 | O(n²) | O(n³) |
| n ≤ 10⁵ | O(n log n) | O(n²) |
| n ≤ 10⁶ | O(n) | O(n log n) |
| n ≤ 10⁹ | O(log n) 或 O(1) | O(n) |

## 附录：刷题心态

1. **第一遍做不出来是正常的**。Hot 100 中的 Medium/Hard 题，大部分人在 LeetCode 上的首次通过率不到 40%。
2. **看懂 ≠ 会写**。必须脱离题解重写一遍。
3. **数量不重要，覆盖度重要**。把 17 个分类各刷 5 题的收获 > 盲目刷 200 题。
4. **面试时的沟通力比写出代码更重要**。学会说出"为什么选这个数据结构"比默默写出 AC 代码更得分。
